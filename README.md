# dev-wiki

업무 중 겪은 **트러블슈팅**과 **CS 처리 기록**, **데일리 노트**를 정리하는 [Obsidian](https://obsidian.md) 보관함.

개념 단위로 잘게 쪼갠 atomic 노트를 `#태그`와 `[[위키링크]]`로 연결하는 **제텔카스텐(Zettelkasten)** 방식으로 관리한다.

## 폴더 구조

| 폴더               | 용도                                                                |
| ------------------ | ------------------------------------------------------------------- |
| `knowledge/`       | 지식 노트 — atomic, 플랫 구조. 개념 1개 = 노트 1개                  |
| `cs/`              | CS 처리 기록 — 현상·원인·처리. 제품 구분은 `#qshop` / `#quv` 태그로 |
| `데일리/`          | 데일리 노트 (`YYYY-MM-DD.md`)                                       |
| `스타일 가이드.md` | 노트 작성 시 따르는 마크다운 스타일 기준 문서                       |
| `CLAUDE.md`        | [Claude Code](https://claude.com/claude-code) 작업 규칙             |

## 작성 규칙

- **노트 1개 = 개념 1개.** 한 덩어리로 길게 쓰지 않고 개념 단위로 분리한다.
- frontmatter `tags`에는 문서를 대표하는 **핵심 키워드**를 단다 (예: `pixel-snapping`). `knowledge`·`settings` 같은 범용 분류 태그는 달지 않는다.
- 새 노트는 관련 노트와 반드시 `[[링크]]`로 연결한다 (고립 노트 금지).
- CS 노트는 처리 기록만으로 충분하며, 기술 개념은 `knowledge/`로 분리하고 `[[링크]]`로 잇는다.

자세한 마크다운 스타일은 [`스타일 가이드`](스타일%20가이드.md) 참고.

## Claude Code

이 보관함은 Claude Code와 함께 관리한다. 주요 스킬:

- `save-notes` — 대화 내용을 atomic 노트로 쪼개 보관함에 저장
- `daily-note` — 오늘 데일리 노트 생성 (이전 팔로업·일정·개발 이슈 자동 반영)
- `commit` — gitmoji 컨벤션에 맞춘 커밋

## 커밋 컨벤션

`<이모지> <타입>: <요약>` 형식의 gitmoji 컨벤션을 따른다.

| 이모지 | 타입   | 사용                      |
| ------ | ------ | ------------------------- |
| 📝     | Docs   | 노트 내용 작성/수정       |
| ➕     | Add    | 새 노트/파일 추가         |
| 🔥     | Remove | 노트/파일 삭제            |
| 🚚     | Move   | 노트 이동/이름 변경       |
| ♻️     | Update | 템플릿/구조 개선          |
| 👷     | Chore  | 설정(`.obsidian` 등) 변경 |
