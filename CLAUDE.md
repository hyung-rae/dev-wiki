# dev-wiki

업무 중 겪은 **트러블슈팅 정리**와 **데일리 노트**를 기록하는 Obsidian 보관함.

## 폴더 구조
- `데일리/` — 데일리 노트 (`YYYY-MM-DD.md`)
- `knowledge/` — 지식 노트 (atomic, 플랫, `#tag` + `[[링크]]`로 연결)
- `cs/` — CS 처리 기록 (제품 구분은 `#qshop` / `#quv` 태그로)

### CS 노트 규칙
- "이 CS를 이렇게 처리했다"는 **처리 기록(현상·원인·처리)만 있어도 충분**하다.
- 기술적인 내용이 포함되면 `knowledge/`에 개념 노트로 분리하고 CS 노트에서 `[[링크]]`로 연결한다.

## 노트 저장 규칙
사용자가 대화 내용을 "노트에 저장해줘"라고 하면 **`save-notes` 스킬**을 따른다.
핵심: 한 덩어리가 아니라 **개념 단위로 쪼갠 atomic 노트 여러 개**를 `knowledge/`에 만들고, 태그 + `[[링크]]`로 연결한다 (제텔카스텐 방식).
- 노트 1개 = 개념 1개.
- frontmatter `tags`에는 그 문서를 대표하는 **핵심 키워드**를 단다 (예: `pixel-snapping`, `subpixel-rendering`). `knowledge`, `settings`처럼 일반적·범용적인 분류 태그는 달지 않는다. 버그 해결 노트에는 `troubleshooting`을 추가할 수 있다.
- 새 노트끼리, 그리고 관련된 기존 노트와 `[[링크]]`로 연결한다 (고립 노트 금지).
- 기존 노트에 이미 있는 개념이면 새로 만들지 말고 보강하거나 링크만 추가한다.

## 스타일 컨벤션
노트를 쓸 때 아래 매핑대로 마크다운 스타일을 **목적에 맞게** 적용한다. 문법 예시는 `[[스타일 가이드]]` 참고.
**원칙: 과용 금지 — 다양하게 쓰는 게 목표가 아니라, 의미가 있을 때만 쓴다.**
- 핵심 결론·중요 단어 → **굵게** (남발 금지)
- 나중에 다시 찾을 핵심 용어 → `==하이라이트==`
- 주의/팁/정보/질문 → 콜아웃 `> [!warning]` `> [!tip]` `> [!info]` `> [!question]`
- 항목·옵션 비교 → 표
- 할 일·상태 → Minimal 체크박스 (`[ ]`할일 `[x]`완료 `[/]`진행중 `[!]`중요 `[?]`확인필요 `[-]`취소)
- 명령·파일명·코드 → `인라인 코드` / 코드블록(**언어 명시**)
- 흐름·관계·구조 → Mermaid 다이어그램
- 개념 연결 → `[[위키링크]]`, 분류 → `#태그`

## 커밋 컨벤션

회사 gitmoji 컨벤션을 따르며, 위키 특성상 아래 타입을 주로 사용한다.

**제목 형식:** `<이모지> <타입>: <요약>`

| 이모지 | 타입 | 사용 |
| --- | --- | --- |
| 📝 | Docs | 노트 내용 작성/수정 |
| ➕ | Add | 새 노트/파일 추가 |
| 🔥 | Remove | 노트/파일 삭제 |
| 🚚 | Move | 노트 이동/이름 변경 |
| ♻️ | Update | 템플릿/구조 개선 |
| 👷 | Chore | 설정(`.obsidian` 등) 변경 |

- 전체 타입 목록은 `/commit` 스킬 참조.
- 본문은 한국어로, **변경 이유**와 **변경 내용**을 서술한다.
- 마지막 줄에 항상 `Co-Authored-By: Claude <noreply@anthropic.com>`를 추가한다.
- 한 커밋에는 하나의 목적만 담는다.

**예시**
```
📝 Docs: Docker 컨테이너 DNS 해석 실패 트러블슈팅 정리

- 컨테이너 내부에서 외부 도메인 해석이 간헐적으로 실패하던 문제 기록
- 원인(systemd-resolved 설정)과 해결 방법, 재발 방지책 추가

Co-Authored-By: Claude <noreply@anthropic.com>
```
