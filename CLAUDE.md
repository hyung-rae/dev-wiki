# dev-wiki

업무 중 겪은 **트러블슈팅 정리**와 **데일리 노트**를 기록하는 Obsidian 보관함.

## 폴더 구조
- `트러블슈팅/` — 트러블슈팅 노트 (플랫, `#tag`로 분류)
- `데일리/` — 데일리 노트 (`YYYY-MM-DD.md`, 데일리 템플릿 자동 적용)
- `knowledge/` — 일반 지식/학습 노트 (atomic, 플랫, `#tag` + `[[링크]]`로 연결)
- `템플릿/` — Obsidian 템플릿 (`트러블슈팅`, `데일리`, `knowledge`)

## 노트 저장 규칙
사용자가 대화 내용을 "노트에 저장해줘"라고 하면 **`save-notes` 스킬**을 따른다.
핵심: 한 덩어리가 아니라 **개념 단위로 쪼갠 atomic 노트 여러 개**를 만들고, 태그 + `[[링크]]`로 연결한다 (제텔카스텐 방식).
- 노트 1개 = 개념 1개. 버그 해결성은 `트러블슈팅/`, 일반 지식은 `knowledge/`.
- frontmatter `tags`에 분류 태그(`knowledge`/`troubleshooting`) + 도메인 태그를 단다.
- 새 노트끼리, 그리고 관련된 기존 노트와 `[[링크]]`로 연결한다 (고립 노트 금지).
- 기존 노트에 이미 있는 개념이면 새로 만들지 말고 보강하거나 링크만 추가한다.

## 커밋 컨벤션

회사 gitmoji 컨벤션을 따르며, 위키 특성상 아래 타입을 주로 사용한다.

**제목 형식:** `<이모지> <타입>: <요약>`

| 이모지 | 타입 | 사용 |
| --- | --- | --- |
| 📝 | Docs | 노트 내용 작성/수정 |
| ➕ | Add | 새 노트/파일 추가 |
| 🔥 | Remove | 노트/파일 삭제 |
| 🚚 | Move | 노트 이동/이름 변경 |
| ♻️ | Update | 템플릿/구조 개선 |
| 👷 | Chore | 설정(`.obsidian` 등) 변경 |

- 전체 타입 목록은 `/commit` 스킬 참조.
- 본문은 한국어로, **변경 이유**와 **변경 내용**을 서술한다.
- 마지막 줄에 항상 `Co-Authored-By: Claude <noreply@anthropic.com>`를 추가한다.
- 한 커밋에는 하나의 목적만 담는다.

**예시**
```
📝 Docs: Docker 컨테이너 DNS 해석 실패 트러블슈팅 정리

- 컨테이너 내부에서 외부 도메인 해석이 간헐적으로 실패하던 문제 기록
- 원인(systemd-resolved 설정)과 해결 방법, 재발 방지책 추가

Co-Authored-By: Claude <noreply@anthropic.com>
```
