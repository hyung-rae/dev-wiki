## 현상

채널톡 CS(2026-06-04): 디자인모드(ED)에서 수정한 **폼 내용이 게시된 사이트(LP)에 반영되지 않고 서로 다르게 보임**. 다시 게시해도 동일. (`whizz.quv.kr/applynow`)

**슬랙 원본**: [#111_voic_큐브-채널톡 스레드](https://squaresinc.slack.com/archives/C04LV3J1CR4/p1780551264456979)

## 원인

큐브 폼은 ==필드 최대 50개== 제한이 있다 — 서브밋 버튼 1개가 포함되므로 실제 입력 필드는 **49개**가 한계.

- **삭제된 항목도 카운트에 포함**된다 — 화면에 보이는 필드가 50개 미만이어도 한도를 초과할 수 있는 비직관적 동작
- 한도 초과 시 별도 에러 안내 없이 게시 반영만 안 되어, 고객에게는 ED/LP 불일치로 보인다

## 처리

- [x] 항목 수를 줄여 폼을 새로 만들거나, 항목이 많이 필요하면 구글폼 사용 안내로 종결

## 유사 사례 — 항목이 너무 길 때도 같은 증상 (2025-10)

필드 **개수**가 아니라 항목 텍스트 **길이**가 원인이었던 건. 폼 저장 시 MySQL 에러 발생:

```text
Row size too large. The maximum row size for the used table type,
not counting BLOBs, is 65535.
```

항목 텍스트가 길면 행 크기 한도(65535 bytes)를 초과해 저장이 깨지고, 역시 ED/LP 불일치로 나타났다. 폼 항목을 짧게 쓰도록 안내. ([스레드](https://squaresinc.slack.com/archives/C04QB59JD1R/p1760066576135609))

> [!tip] 진단 포인트
> **폼이 ED와 LP에서 다르게 보이면 폼 저장 한도부터 의심**한다 — ① 필드 수 50개(삭제 항목 포함) ② 항목 길이(row size). 두 경우 모두 사용자에게 에러가 노출되지 않는다.
