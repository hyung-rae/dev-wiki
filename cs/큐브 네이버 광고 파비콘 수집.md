## 현상

고객이 **파비콘 파일을 수정**했는데, **네이버 광고 측에서 변경된 이미지를 수집하지 못하는** 현상.
- 서버의 파비콘 파일은 새 이미지로 바뀐 상태
- 그러나 네이버 광고에는 계속 이전 파비콘이 노출됨

## 원인

==파비콘 URL이 그대로라 네이버 광고 측이 같은 파일로 인식==해 재수집하지 않은 것.

→ 근본 개념: [[캐시 무효화 - URL 기반 캐싱]]

### 큐브의 파비콘 `<link>` 생성 로직

마스터 코드에서 사이트별(`$stid`)로 파일 존재 여부를 확인해 우선순위대로 `<link>` 태그를 출력한다. (`symbolic/pub/inc/header.inc.html:391-399`)

1. `favicon_96.png`가 있으면 → `<link rel="icon">` + `apple-touch-icon` (96×96)
2. 없고 `favicon.ico`가 있으면 → `<link rel="shortcut icon">`
3. 둘 다 없으면 → 기본 파비콘 (`https://quv.kr/img/ico/qv_favicon.ico`)

각 링크에는 캐시 무효화용 `?v=<?=$time?>` 파라미터가 붙는다.

> [!info]
> 이 구조상 `favicon_96.png`를 그대로 두고 내용만 교체하면 URL(파일명)이 동일해, 외부 수집기는 변경을 인지하지 못한다.

## 처리

- [x] 기존 `png` 파일을 제거하고 `ico` 타입으로 파비콘을 새로 등록 → **파일명이 바뀌면서 URL도 변경됨**
- [/] 네이버 광고 측 수집 결과 대기 중
