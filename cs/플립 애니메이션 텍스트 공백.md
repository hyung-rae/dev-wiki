## 현상

quv 퍼블리시 페이지(`quvcsfront.quv.kr`)에서 **flip(`flip-x`/`flip-y`) 애니메이션 텍스트 블록이 간헐적으로 공백**으로 뜬다. 애니메이션이 안 보일 뿐 아니라 **원래 텍스트조차 아예 안 보이고**, **화면을 클릭하면 그제야 노출**된다(2026-06-29).

- 안 보이는 순간에도 `opacity:1`, `transform` 단위행렬, 애니메이션 `finished` — **computed style 은 정상인데 픽셀만 공백**
- ==DevTools 를 켜면 정상== 동작 → [[DevTools를 열면 사라지는 렌더링 버그]]
- `fade`/`scale`/`translate` 계열은 멀쩡하고 **flip 에서만** 발생

## 원인

flip 의 ==3D 회전(`rotateX/Y(90deg)`)== 이 90° 에서 요소를 **0px 로 투영**시키고, `translateZ(0)`+`perspective` 로 승격된 GPU 합성 레이어가 **빈 텍스처로 캐시**된 뒤 정면(0°) 복귀해도 재래스터화되지 않아 공백으로 남는 것. → [[GPU 합성 레이어와 빈 텍스처]]

- flip 만 0px 로 퇴화하므로 다른 애니메이션은 멀쩡 → [[CSS 2D transform vs 3D transform 래스터화 차이]]
- 클릭은 리페인트를 유발해 빈 텍스처를 다시 그리게 만들어 노출시킨 것

## 처리

`symbolic/pub/css/animate.css` 의 flip keyframe 을 ==각도 제한 3D== 로 변경. 시작 각도를 90°→**60°**로 낮춰 투영 높이가 50% 로 유지되게 해(0px 회피) 빈 텍스처를 원천 차단하고, `perspective()` 로 3D 입체감은 유지한다.

```css
/* 기존: 0%{rotateX(90deg)} → 90°에서 0px → 공백 */
@keyframes ani_flip_x {
  0%   { transform: perspective(600px) rotateX(60deg); opacity: 0; }
  100% { transform: perspective(600px) rotateX(0deg);  opacity: 1; }
}
@keyframes ani_flip_y { /* rotateY 로 동일 */ }
```

- [x] flip keyframe 을 `perspective(600px) rotateX/Y(60deg→0deg)` 로 교체 (핵심)
- [x] 로컬 비교 테스트 HTML 로 **DevTools 닫고** 공백 없음 + 3D 느낌 육안 검증
- [-] 2D `scaleY/X` 대체안 — 버그는 없으나 입체감이 사라져 폐기
- [-] `backface-visibility:hidden`·keyframe `perspective()` 복원·JS 강제 재래스터화 — 효과 없거나 불필요해 폐기

> [!tip] 교훈
> "에디터·DevTools 에선 멀쩡한데 실제 화면에서만 콘텐츠가 공백"이고 **클릭하면 보이면**, GPU 합성 빈 텍스처를 의심한다. 3D 회전이 90°(0px)에 닿는 게 원인이면 **각도만 60° 정도로 제한**해도 입체감을 지키며 해결된다. 검증은 반드시 DevTools 닫고.
