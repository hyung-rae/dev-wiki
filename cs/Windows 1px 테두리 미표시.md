## 현상

qshop editor에서 **상품 슬라이드 블록**에 상품 테두리를 적용했을 때, **Windows 환경에서만 좌측 테두리가 보이지 않는** 현상 발생.
- 뷰포트를 줄였을 때 border가 렌더링되지 않거나 사라짐
- macOS에서는 정상 표시, Windows에서만 재현

**노션 원본**: [상품 슬라이드 블록 - 상품 테두리 적용 후 윈도우에서만 좌측 테두리가 보이지 않는 현상](https://www.notion.so/squares-inc/373311a8a9b080c8a6f1cd013769a538)

## 원인

**브라우저 서브 픽셀 렌더링 버그** — CSS 1px 보더가 Windows DPI 배율(125%, 150%)로 인해 소수점 픽셀 값이 되고, 브라우저가 이를 0px로 내림하여 미표시.

→ 기술적 원인 상세: [[CSS 1px 테두리 렌더링 이슈]]

## 비개발자 설명 (공유용 문구)

> "컴퓨터가 화면 크기를 계산하다가 생긴 소수점 단위를 Windows OS가 버림 처리(0으로 계산)하면서 선이 사라지는 현상입니다."
> - 모니터 점은 0.5개를 켤 수 없어 무조건 1 또는 0으로 처리됩니다.
> - 3등분(33.3%)이나 화면 줄이기/늘리기 시 내부에 소수점(예: 0.4픽셀)이 생깁니다.
> - Mac은 소수점이 생겨도 흐릿하게라도 선을 그려주지만, Windows는 칼같이 잘라냅니다.
> - **대책**: 소수점 버림 오류에 영향받지 않는 방식(그림자 효과 우회)으로 코드를 수정합니다.

## 재발 방지

상품 테두리 등 **1px 외곽선**을 구현할 때는 `border` 대신 `box-shadow: 0 0 0 1px [color]` 사용.
