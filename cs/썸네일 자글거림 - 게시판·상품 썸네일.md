## 현상

채널톡 CS(2026-06-05): **게시판 대표이미지(썸네일)가 자글자글 깨져 보인다**는 문의.
- 고객이 썸네일 비율 설정을 건드린 적 없고, 권장 크기 이미지로 테스트해도 동일 재현
- 특정 사이트만이 아니라 가이드 사이트(`qshop-guide.qshop.ai`)에서도 재현 — **ED, LP 모두 발생**

## 원인

같은 시기 FE 변경·배포가 없었는데 증상이 시작됨 → 코드 회귀가 아니라 **브라우저 업데이트로 인한 회귀**.

- 썸네일에 적용돼 있던 ==`image-rendering`== CSS 값이 원인 — 해당 속성을 끄면 정상 표시
- 그동안 Chrome이 이 값을 **적용하지 않다가 Chrome 149 업데이트부터 실제 적용**되면서, 축소 렌더링되는 썸네일에 계단 현상(자글거림)이 나타난 것으로 파악 ([Chrome 149 베타 노트](https://developer.chrome.com/blog/chrome-149-beta?hl=ko))

→ 속성 동작 상세: [[CSS image-rendering 속성]]

## 처리

- [x] `image-rendering` 속성 변경(제거)으로 해결 — **게시판 썸네일 + 상품 썸네일 함께 수정**
- [x] dev 배포 후 QA — 자글거림 없음, 상품 등록 시에도 정상 확인 (2026-06-08)
- [x] 정기배포 (2026-06-09)

> [!tip] 교훈
> 배포한 게 없는데 갑자기 생긴 렌더링 이슈는 **브라우저 버전 업데이트(릴리스 노트)부터 의심**한다. 브라우저가 무시하던 CSS 값이 업데이트로 갑자기 적용되며 회귀처럼 나타날 수 있다.
