같은 합성 레이어라도 ==2D 변형(`scale`·`translate`)== 과 ==3D 변형(`rotateX/Y`·`perspective`)== 은 **텍스처를 래스터화하는 방식이 달라서**, 3D 회전만 "0px 퇴화 → 빈 텍스처" 버그에 걸린다.

## 핵심 차이

| 구분 | 래스터화 방식 | 0px 상태일 때 |
|---|---|---|
| **2D `scaleY(0)`** | 콘텐츠를 **자연 크기로 1회** 래스터화하고, 합성기가 그 텍스처를 스케일만 적용 | 텍스처엔 콘텐츠가 그대로 있고 높이만 0 → 커지면 정상 복원 |
| **3D `rotateX(90deg)`** | **투영된 크기**로 래스터화 | 투영 0px → 빈 텍스처가 캐시됨 → [[GPU 합성 레이어와 빈 텍스처]] |

## 왜 scale-up 은 멀쩡한데 flip 은 터지나

- `scale-up`(scale `0.3→1`): 시작이 30% 라 0px 가 아니고, 2D 라 텍스처는 항상 풀 콘텐츠 → 절대 공백 없음.
- `flip`(rotateX/Y `90→0deg`): 90° 에서 정확히 0px 로 투영 + 3D 라 투영 크기로 래스터화 → 빈 텍스처.

> [!tip] 실무 판단
> 펼침/등장 효과에서 합성 버그가 의심되면, **3D 회전을 2D 스케일로 바꾸거나**(입체감을 포기할 수 있을 때) **회전 각도가 0px(90°)에 닿지 않게** 제한한다. 2D 변형은 0 에서 시작해도 안전하다.
