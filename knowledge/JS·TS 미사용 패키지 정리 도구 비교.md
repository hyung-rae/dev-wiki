JS/TS 프로젝트의 미사용·누락 패키지를 정리하는 도구는 크게 **Knip / depcheck / npm-check** 셋이다. 핵심: **Knip이 depcheck의 상위 호환**이고, npm-check는 버전 업데이트까지 챙기는 대화형 관리용이다.

## 세 도구 비교
| 도구 | 주 목적 | 장점 | 단점 / 현황 |
| --- | --- | --- | --- |
| **Knip** | 데드 코드 + 미사용 파일 + 패키지 종합 | 최신 TS/모노레포 지원, 높은 정확도, `export`까지 추적 | 설정 옵션 많아 초기 학습 필요 |
| **depcheck** | `package.json` 패키지 분석 | 가볍고 단순, 높은 인지도 | **업데이트 중단**, 최신 문법 오탐 많음 |
| **npm-check** | 패키지 정리 + 버전 업데이트 | 대화형 터미널 UI로 관리 편리 | 코드 수준(함수·파일) 분석 불가 |

## depcheck — 전통의 강자
`package.json`의 미사용·누락 패키지를 찾는 오랜 표준. 다만 유지보수가 멈춘 상태라 최신 TypeScript 문법·모노레포에서 **멀쩡히 쓰는 패키지를 안 쓴다고 오탐**하는 일이 잦다. → 이를 극복한 상위 호환이 [[Knip - 데드 코드·의존성 청소 도구|Knip]].

## npm-check — 대화형 관리
미사용 확인뿐 아니라 업데이트가 필요한 패키지까지 보여준다. `-u` 옵션이면 화살표·스페이스바로 직관적 삭제/업데이트가 가능하다.
```bash
npx npm-check -u
```

## 선택 가이드
- [!] **대형·프로덕션** → Knip 도입 + CI 연동.
- [!] **소규모·토이** → `npm-check -u`로 가볍게 버전 올리고 수동 정리.
