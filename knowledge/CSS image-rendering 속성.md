`image-rendering`은 이미지가 원본과 다른 크기로 렌더링될 때 **브라우저가 어떤 보간(interpolation) 알고리즘으로 스케일링할지** 지정하는 CSS 속성이다.

| 값 | 동작 | 용도 |
| --- | --- | --- |
| `auto` | 브라우저 기본 — 부드러운 보간 | 일반 이미지 (기본값) |
| `smooth` / `high-quality` | bilinear 등 부드러운 보간을 명시 | 사진, 썸네일 |
| `crisp-edges` | 보간 최소화 — 경계를 선명하게 유지 | 라인아트, QR코드 |
| `pixelated` | nearest-neighbor — 픽셀을 그대로 확대 | 픽셀아트 |

핵심: `crisp-edges`·`pixelated` 계열은 안티앨리어싱 없이 스케일링하므로, **사진·썸네일을 축소 렌더링하면 자글자글한 계단 현상**이 생긴다.

> [!warning] 브라우저 지원이 균일하지 않다
> 값별 지원·해석이 브라우저마다 다르고, **무시되던 값이 브라우저 업데이트로 갑자기 적용**될 수 있다. 이 경우 코드 변경 없이 렌더링 회귀가 발생한다 — 실제 사례: Chrome 149부터 적용이 바뀌며 썸네일이 깨진 [[썸네일 자글거림 - 게시판·상품 썸네일]].
